## 2. 设计目标

### 2.1 核心原则

1. **零依赖部署**：所有前端资源内嵌在代理中，`npm install -g` 即可用，无需额外安装
2. **单文件架构**：延续当前单 HTML 文件模式，不引入构建工具链
3. **新手友好**：向导模式让非技术人员也能完成配置
4. **专业可用**：高级用户可直接操作所有细节

### 2.2 用户角色

| 角色 | 场景 | 需求 |
|------|------|------|
| **自助用户** | 自己用 | 监控状态、切换模型、查看用量 |
| **帮助者** | 帮别人配 | 一键向导、远程诊断、配置导出 |
| **被帮助者** | 被配置 | 简单易懂的状态页、问题提示 |

## 5. 技术方案

### 5.1 架构选型对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A. 增强单文件 HTML**（当前） | 零依赖、部署简单、加载快 | 代码量大、维护难度中 | ⭐⭐⭐⭐ |
| **B. Vue/React SPA** | 组件化、生态丰富 | 需要构建工具、部署复杂 | ⭐⭐⭐ |
| **C. 渐进式增强** | 先用方案A，图标/图表按需加载 CDN | 平衡 | ⭐⭐⭐⭐⭐ |

### 5.2 推荐方案：渐进式增强（方案 C）

**核心栈**：
- **框架**：Alpine.js（已使用，轻量，无需构建）
- **CSS**：Tailwind CSS CDN（已使用）
- **图表**：Chart.js CDN（按需引入，~60KB gzip）
- **图标**：Lucide Icons（内联 SVG，按需使用）
- **实时更新**：EventSource（SSE，代理新增 `/v1/events` 端点推送状态）

**文件结构**：
```
lib/
├── admin.html          # 主页面（增强为多 Tab SPA）
├── admin-page.js       # 路由逻辑（新增，负责加载 HTML）
├── server.js           # 后端 API
└── converter.js        # 协议转换
```

### 6.2 新增接口

#### `GET /v1/events`
SSE 实时事件流。客户端：
```javascript
const evt = new EventSource('/v1/events');
evt.addEventListener('status', e => updateDashboard(JSON.parse(e.data)));
evt.addEventListener('request', e => addToLog(JSON.parse(e.data)));
evt.addEventListener('error', e => showAlert(JSON.parse(e.data)));
```

│   ├── 安装 undici 依赖
│   ├── 部署代理增强补丁
│   ├── 配置 launchd 开机自启

├── patches/                    # 代理增强补丁
